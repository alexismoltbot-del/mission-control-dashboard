# GEO Agency - QA Gates

The product is not client-ready if any of these fail.

## Gate 1 - Offer clarity

- pricing is visible
- deliverables are explicit
- what is not included is explicit

## Gate 2 - Trust

- no unsupported claim
- QA and human review are visible
- support path is visible

## Gate 3 - Conversion path

- one primary CTA
- no dead-end after CTA
- checkout and intake path are coherent

## Gate 4 - Auth and care

- auth path works
- client can find support
- onboarding path exists

## Gate 5 - Technical hygiene

- analytics events are mapped
- basic mobile review is done
- launch checklist is green

## Gate 6 - A/B readiness

- first 2 tests are defined
- metrics are chosen
- event mapping exists

## Release rule

If a gate fails, launch is blocked until the failure is fixed or explicitly
accepted by Alexis.
