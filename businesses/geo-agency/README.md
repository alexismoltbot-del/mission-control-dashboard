# GEO Agency

## Role in the factory

GEO Agency is the service layer that helps client sites become easier to
discover, parse, cite, and trust across AI search surfaces.

This is not a generic SEO shop.
This is a focused GEO/GEO-adjacent offer for LLM visibility, entity clarity,
and machine-readable trust.

## Core promise

> We turn a client website into a source AI systems can index, understand, and
> cite with confidence.

## Objective

Make the offer client-ready so Alexis can take paid clients without hiding the
rough edges behind custom manual work.

## Definition of done - client-ready

- premium marketing site structure exists
- pricing is clear and defensible
- one dominant CTA path exists
- auth path for client access is defined
- payment plus intake path is defined
- customer care entrypoint is visible
- QA and launch gates are written
- first 2 A/B tests are defined and instrumentable
- launch checklist exists

## What "A/B tested" means in phase 1

Not "proven by large sample size" yet.

It means:

- 2 clear variants exist
- success metrics are defined
- analytics events are mapped
- the first experiment can go live without rethinking the site

## Main workstreams

1. Offer and pricing
2. Site and conversion path
3. Auth, care, and implementation stack
4. QA, launch, and go/no-go discipline
5. A/B testing and KPI instrumentation

## Constraints

- no fake proof
- no overclaim on LLM rankings
- no launch until support, pricing, and QA are coherent
- phase 1 optimizes for first paying clients, not scale
